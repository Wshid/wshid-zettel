기술 분야에서의 접근성(accessibility)

다이어그램을 만들 때는 독자가 완전히 접근할 수 있는지 고려 필요
- 이해관계자가 메시지를 이해하지 못하면 양쪽 모두 손해

다이어그램과 시각 자료는 지식 수준, 비즈니스 기능, 제품 또는 도메인에 대한 친숙도가 천차만별인 다양한 사람들이 접근 가능해야 함
- 사용 중인 화면 크기와 정보를 소비하는 시간 등 대상 고객이 처한 상황도 다이어그램 접근성에 영향을 미침

더 많은 사람이 접근할 수 있는 다이어그램을 만드는 방법

### NOTE
누구나에게 장애가 생길 수 있음
- 장애를 갖게 만드는 것은 그들의 환경
- 여러분의 목표는 모든 사람이 동등한 위치에 서도록 하는 것

# 색상에 의존한 커뮤니케이션
- 색상에 의존한 커뮤니 케이션(relying on color to communicate)는 의미를 나타내기 위해 색상만을 사용하는 것
	- e.g. 다이어그램의 새롭거나 변경된 항목을 나타내기 위해 다야한 색상의 상자 사용
- 가장 분명하고 일반적인 형태 -> 색상을 구분할 수 없는 사람들을 위한 별도의 추가 설명(식별표) 없이 색상 사용하여 긍정 부정 구분
	- 별도의 표시가 없어도 녹색은 '좋음', '진행'
	- 빨간생은 '나쁨', '정지'
- 신호등은 적어도 위치를 색상 의미의 지표로 활용하나
	- 다이어그램에서는 빨간색, 초록색은 이러한 단서 없이 사용되는 경우가 많음

빨간색과 초록색을 벗어나려는 움직임
- e.g. github내 diff
	- 추가는 초록색 더하기
	- 삭제는 빨간색 빼기 기호 표시
- 다른 색상에서는 이러한 것이 없기 때문에 다이어그램 및 기타 시각 자료 접근성에 영향

### NOTE
- Event Storming(협업 모델링 기법)에서는
- 서로 다른 요소를 표현하기 위해 색상 스티커 메모 사용
- 이 메모는 모든 사람이 두요소의 차이를 알 수 있도록 모양이 다름

모든 사람이 색을 똑같이 인식한다고 볼 수 없음
- 인구의 4.5%가 색맹(색각 이상, color vision deficiency)
- 여성(1/200명), 남성(1/12명)
- 시각 자료에서 두 가지 이상의 색상 차이를 인식하지 못할 가능성이 큼

색맹 외에도 부분 시력을 가진 사람들처럼 낮은 대비를 인식하는데 어려움을 겪는 경우도 있음
- [[광 공포증]], [[원추각막]], [[녹내장]] 등의 질환도 다이어그램과 그 색상을 얼마나 잘 볼수 있는지에 영향 미침
- 색상뿐만 아니라 [[대비]]도 고려해야 다이어그램에 대한 접근성을 높일 수 있음

다이어그램의 색상에 대한 인식을 영향을 미치는 또 다른 요소, 포맷
- 독자가 다이어그램을 보는데 사용하는 모니터나 프로젝터는 다이어그램을 만든 모니터와 매우 다르게 보정될 수 있음
- 비슷한 색상이 구별되지 않거나 식별할 수 없을 수 있음
- 독자에게는 노란색으로 보이는데, '녹색'이라고 언급하면 독자는 혼란스러울 것

### TIP
위키/웹 페이지를 생성하거나 프로젝트를 사용해 프레젠테이션을 할 때와 같이
- 여러 매체에서 다이어그램을 활용하는 상황에서는 서로 다른 컬러 팔레트 사용 가능
- 웹 페이지의 배경색이 흰색이고, 슬라이드 배경색이 검은색인 경우와 같이 매체마다 배경색이 다른 경우 특이 그럼
- 독자가 색상을 어떻게 인식하는지, 배경색이 요소와 레이블의 명암비를 어떻게 변화시키는지 고려할 것

다이어그램이나 시각 자료는 문서(종이, 팜플렛 등)나 책과 같이 인쇄된 형태로도 볼 수 있어 그레이스케일로 인쇄될 수 있음
- 그레이스케일에서는 모든 색상이 사라지고 채도(밝음/어두움)만 남음
- 이 경우 다이어그램의 모든 색상이 동일하게 보일 수 있음

색상을 구분할 수 없으면 전달되는 의미가 사라짐
- 독자가 색을 구분할 수 없다면 각 색이 무엇을 의미하는지 보여주는 식별표는 의미가 x
- 다이어그램을 제시하고 색상으로 구성 요소를 언급할때, 다이어그램이 색상인지 그레이스케일인지에 따라
	- 어떤 구성 요소를 언급하고 있는지 알 수 없는 독자도 있을 것
- 눈에 보기 좋다고 해서 메세지가 성공적으로 전달될 것이라고 속단 x

**색상에 의존한 커뮤니케이션 안티패턴**을 피하면서 독자에게 색상 외 다른 구분법을 제고할 수 있음
- 패턴을 사용하는 것
- 적절하게 대비되는 패턴(e.g. 단색 막대, 줄무늬 막대)을 사용하면 색맹도 그레이스케일을 보고 색상 쉽게 구분 가능
- 이는 다이어그램의 컬러 팔레트를 다양화 할 수 없을 때 유용함

말이나 글로 다이어그램을 언급할 때 주의할점 -> '다이어그램의 빨간색 상자'와 같이 색상을 기준으로 언급하지 않는 것
- 이를 '점선으로 그려진 빨간색 상자'와 같이 표현할 수 있음

기호는 구성 요소나 색상을 구분할 때 유용한 또 다른 도구
- 단독으로 사용되거나 색상과 함께 사용 가능
- e.g. 빨간색, 노란색, 녹색을 상태에 사용할 때 녹색에는 +, 빨간색에는 -를 추가하는 등
	- **기호를 색상과 함께 사용**하면 메세지를 더 명확하게 만들 수 있음

### CAUTION
다이어그램에 기호를 사용하기 전에 문화적 측면에서 기호가 어떤 의미를 가지고 있는지 찾아봐야 함
- 의도한 메세지를 문제없이 전달할 수 있는지, 또는 혼란이나 불쾌감을 주지 않는지 확인해야 함

편하게 접근성 컬러 팔레트 생성도 가능
- 어떤 컬러 팔레트가 접근성이 있는지는 도구를 사용해 쉽게 확인 가능
- 다수의 무료 도구들이 있으며, 일부는 브라우저 익스텐션으로 제공되어 웹 사이트 팔레트 확인시 용이

### 색맹을 위한 디자인 도구
접근성 컬러 팔레트 생성시 도움이 되는 도구
- Color Oracle
	- 4가지 유형의 색맹(그레이스케일/단색 포함)을 시뮬레이션 할 수 있는 윈도우, 맥, 리눅스용 무료 어플리케이션
- Coblis
	- 이미지를 업로드하고 8가지 유형의 색맹을 시뮬레이션 할 수 있는 무료 온라인 도구
- Sim Daltonism
	- 8가지 색맹 유형을 시뮬레이션
	- 맥에서는 드래그 및 크기 조정 가능한 창이 존재
		- 맥이나 아이폰을 사용할 경우 Color Oracle보다 더 편하게 사용 가능
- Chromatic Vision Simulator
	- 안드로이드 및 iOS용 무료 어플리케이션이며 웹 버전도 있음
	- 휴대폰 카메라에서 실시간으로 또는 저장된 사진 이미지를 사용하여 최대 세 가지 유형의 색맹 재현
- Viz Palette
	- 데이터 및 기타 시각 자료의 컬러 팔레트 선택하는 무료 도구
	- 혼동될 수 있는 색상 리포트를 제공하며, 자바 스크립트에서 복사/붙여넣기 하는데 최적화


draw.io의 표준 색상중 일부
- 여러가지 팔레트를 제공하면서, 사용자가 직접 사용 선택 가능
- 일부 독자 배제가 가능하지 때문에, 기본 팔레트가 가장 접근성 좋다 생각하고 사용해서는 안됨

녹색맹이 있는 사람을 위한 시뮬레이션
- 다이어그램에는 4가지 색상이 사용되었음. 얼핏 보기엔 3색상
- 컬러 팔레트를 고려할때는 시뮬레이터 도구 뿐만 아니라 채도(밝기/어두움)가 다른 색상을 포함하는게 좋음
- 높은 대비가 필요한 사람 or 색맹, 색상 간의 대비가 차별성을 가져옴

다이어그램 어플리케이션이나 기업의 색 구성표를 사용할 때는 그레이스케일 또는 색맹과의 호환성 확인 필요
- 디자이너와 회사(특히 예산이 많은 대기업)가 공식 색상표, 앱의 기본 색상 선택시 접근성을 고려했다고 믿을 수 있으면 좋겠으나, 그렇지 않은 경우가 많음
- 색맹 시뮬레이터를 사용하여 회사 컬러 팔레트 또는 자주 사용하는 다이어그램 생성 애플리케이션의 기본 색상 테스트 할 것
- 좋은 점이든 나쁜 점이든 피드백을 제공하면 좋음

### TIP
- 대체 텍스트(alternative text, alt text)는 이미지 또는 다이어그램에 대한 텍스트 설명 제공
- 특히 사용자를 위한 문서 같은 공개 컨텐츠의 경우 다이어그램이나 기타 시각 자료에 대한 대체 텍스트를 신중하게 작성해야 함


특히 색맹이나 장애가 있는 사람들로부터 다이어그램에 대한 피드백을 받고
- 이를 다시 다이어그램과 향후 다른 시각자료에 반영할 것
