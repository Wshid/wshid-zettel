# 색의 과부하
색의 과부하 안티패턴(color overload antipattern) -> 유니콘의 폭발(Explosion of Unicorn)
- 일반적으로 다이어그램에 색을 많이 사용하지 않거나, 전혀 사용하지 않기도 함
- 많은 색을 사용할 경우 독자가 색상 - 의미 연관을 위한 큰 노력이 필요

이러한 안티패턴 -> 시각적 커뮤니케이션에서 색상이 중요하다는 사실 간과
- 다이어그램 작성자는 애플리케이션에서 제공하는 기본 색상(또는 무작위 색상)을 사용하는게 편거나, 익숙한 색 사용

### TIP
다이어그램을 만드는 시간을 아끼는 것은 '호미로 막을 것을 가래로 막는'(penny-wise but pound-foolish)격
- 다이어그램을 다시 만들고, 잘못된 의사소통으로 인한 혼란을 수습하는데 더 많은 비용이 소요됨
- 코드의 버그를 조기에 수정하는 것과 마찬가지로, 커뮤니케이션을 조기에 바로잡으면 장기적으로 비용, 시간 절약 가능

이 안티패턴은 다이어그램 뿐 아니라 흑백으로만 만들어진 것을 제외한 모든 시각 자료에 해당
- 하지만 모든 시각 자료를 흑백으로 만들라는 것이 아님
- 안티패턴 극복 방법: 커뮤니케이션에 사용할 색상을 고려하는 것

각 구성요소가 서로 다르다는 것을 보여주기 위해 채도가 높은 색상을 많이 사용할 경우
- 이 문제를 해결하려면 컬러 팔레트(color palette)를 최소화 하고 메세지를 전달하는 데 필요한 색상만을 사용해야 함
- 다이어그램 모든 구성 요소에 다른 색상 사용 필요 x
- 색상을 선택할 때는 어떤 색상이 서로 잘 어울리는지(= 서로 충돌하지는 않는지), 색상 밝기도 고려해야 함
- 너무 밝은 색상은 독자에게 부담됨

색상을 선택하는 것 외에도 선택한 색상을 통해 무엇을 전달할 것인가
- 기능 공유하는 구성 요소에 동일 색상 사용, 함수나 유형 같은 것 표현
- 색상의 의미를 나타내기 위한 식별표를 차트 하단에 위치
- 다이어그램이 산만해지는 것을 최소화
- 메세지가 산만해지는 것을 최소화하는 것이 성공적인 커뮤니케이션의 핵심

# 상자 속 상자 속 상자
상자는 다이어그램에서 구성 요소의 위치(개념적, 논리적, 물리적)을 전달하는데 자주 사용됨
- 구성 요소의 그룹화를 위한 목적으로 사용
- 상자 과도 사용 -> 선이 많아져 혼란스러움
- 다이어그램의 가독성을 위한 충분한 공백을 확보하는 데에서 많은 공간이 필요하기 때문에 실제 내용 전달을 위한 공간이 줄어듦

상자 속 상자 속 상자 안티패턴(boxes in boxes in boxes antipattern)은 다이어그램 작성자가 다른 내용을 동일한 형태의 상자로 표현할 때 나타남
- 의미를 표현하는 다른 방법을 알지 못하거나 '항상 그래왔던 방식' 사고방식에 갇힐 수 있음

### TIP
공백(whitespace)는 다이어그램의 내용만큼이나 중요
- 공백은 눈이 쉴 수 있는 공간을 제공하여 인지 부하를 줄여줌
- 다이어그램을 더 쉽게 스캔할 수 있게 해주고 가독성을 높여줌

원인이 무엇이든 간에 이러한 문제는 의사소통 방해 -> 시간, 비용 낭비
- 독자는 다이어그램 이해 노력 좋아하지 x
- 노력을 기울이지 않거나, 노력하더라도 의도한 메세지 이해 x 가능성 존재

가장 흔히 발생하는 안티패턴, 상황이나 위치(e.g. 논리적 위치)를 전달하는 구조 다이어그램
- 데이터 흐름도나 sequence diagram에서도 선, 상자가 너무 가까우면 문제 발생 가능

### TIP
배경색 사용시 테두리나 텍스트보다 차분한 색 사용
- 대비를 유지하면서 다이어그램의 배경색과의 차이 유도 가능

다이어그램을 만들 때는 상자 이외의 다른 표현 방법도 고려해야 함
- 컴포넌트에 레이블을 붙이는 등
- 상자를 사용하는 경우 상자의 윤곽선과 배경의 색상 및 패턴 사용 구분
- 불필요한 세부사장 제거 및 다이어그램 분할 등

개선 방향
- 일부 상자를 레이블이나 메모로 대처하기
- 메세지 전달을 위해 별도 분리할 필요가 없다면 상자 병합
- 남아 있던 상자는 색상, 패턴(실선과 색상 배경)을 사용하여 구분

상자는 결국, 다이어그램에서 의미를 전달하는 한 가지 방법
- 상자를 중첩 사용시 시각적으로 혼란스러울 수 있음
- 공백은 이 안티패턴에서 설명한 다른 전략과 함께 활용하면 좋음
