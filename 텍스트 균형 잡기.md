모든 유형의 정보가 다이어그램으로 전달하기 적함한 것 x
- 오히려 너무 많은 정보가 있으면 의도한 메세지가 모호해짐
- 때로는 의도한 메세지를 전달하기 위해 다이어그램에 정보 추가하는 경우 존재
- 그러한 경우 **텍스트 균형 잡기 패턴**(balance text pattern)을 적용
- 독자가 메세지를 이해할 수 있도록 충분한 정보를 제공하는 것
	- 다이어그램이 너무 복잡하여 메세지가 손실되거나 부분적으로 전달되는 것 사이에서 균형을 맞추기

### TIP
일부 정보는 다이어그램에 표시하기 적합하지 않으므로,
- 다이어그램과 함께 텍스트 또는 표 형식으로 이동해야 함
- 문장으로 작성된 정보는 축약 or 다이어그램에서 제거 -> 별도의 텍스트로 만들기
- 관계형 데이터는 표 형식으로 하기 좋음

흐름도 다이어그램(flow diagram)과 같은 일부 표기법과 다이어그램 중에는 시각적으로 혼란스럽지 않게
- 텍스트를 추가할 수 있는 표준 방식 내장
- 다른 다이어그램의 경우 메모나 각주를 사용하면 기본 가이어그램을 어지럽히지 않고 더 많은 정보 추가 가능

### CAUTION
메모를 사용한다면 다이어그램 옆에 텍스트를 적든지 해서 완전히 분리되는 것에 주의
- 메모가 없어지거나 메모 없이 다이어그램을 볼 경우 독자가 오해할 수 있음

### 예시
다이어그램에서 '고객'과 같이 불필요하게 반복되는 텍스트는 완전히 제거하거나 줄이기
- 필요한 정보는 흐름도 주석이나 4개의 숫자메모(다이어그램에 위첨자 참조)를 배치하여 주요 흐름을 설명
- 다이어그램 박스 내의 텍스트는 최대한 줄이는게 목적
	- 과도한 텍스트는 제거하거나 주석 및 참조로 옮기기
불필요하게 길어진 문장을 핵심만 전달하도록 짧은 문장으로
- 제목과 문맥을 통해서도 주어인 '고객' 유추는 가능함
- 흐름도 자체에서 유추할 수 있는 정보 반복 x

기본 다이어그램에서 제거된 정보를 표시하는 방법은 다이어그램 유형,
- 전달하려는 메세지 및 전달 형식(프레젠테이션, 문서 또는 기타 형식)에 따라 다름

불필요한 정보를 모두 메모나 각주로 옮기고 싶은 유혹을 느낄 수 있지만
- 이는 문제를 해결하는 것이 아닌 치워두는 것
- 독자는 여전히 과도한 정보를 해쳐 나가야하는 불편을 겪을 것
- 메세지 전달에 필요하지 않은 정보는 모두 제거할 것
- 필요한 경우 다른 다이어그램으로 옮길 수 있음
