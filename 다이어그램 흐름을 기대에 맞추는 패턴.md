다이어그램을 만들 때 독자가 어떻게 읽을지 고려하지 않는 경우가 많음
- 모든 다이어그램에는 구조, 동작을 전달하는 정보의 흐름이 있으며,
	- 이를 독자가 다이어그램을 읽는 방식에 최대한 가깝게 일치시켜야 함

책을 볼때 텍스트가 왼쪽 상단 -> 오른쪽 하단 으로 예상
- 다이어그램의 경우에도 그래야 함
- 성공적인 커뮤니케이션: 발표자와 청중 사이의 커뮤니케이션 장벽을 가능한 한 많이 허무는 것

다이어그램의 초점이나 시작 부분을 다른 곳에서 시작하는 것보다
- 왼쪽 상단 | 가운데에 두는 것이 훨씬 더 가독성이 좋음
- 이후 위 -> 아래, 왼쪽 -> 오른쪽의 일반적인 흐름을 따라가면 독자가 다이어그램 메세지를 이해하는데 도움이 됨

### TIP
편집하거나, 재정렬할 수 없는 다이어그램인 경우
- 혼란을 피하기 위해 다이어그램 읽기 시작하는 위치에 레이블이나 기호 추가 가능
	- **여기에서 시작** 문구 | 화살표 | 손가락 | 재생버튼 같은 기호 사용 가능
- 번호가 매겨진 레이블을 사용하여 올바른 순서로 다이어그램을 따라가도록 할 수 있음

### TIP
이야기와 마찬가지로 다이어그램에는 순서가 있어야 함
- 시작: 독자가 읽기 시작해야 하는 부분
- 중간: 적절한 순서로 정렬된 나머지 컨텐츠
- 끝: 독자가 도출하기 위한 결론

시각적으로 고려해야 할 또 다른 측면, 흐름도가 묘사하는 상호작용(레이블이 붙은 관계 화살표)
- **요청은 텍스트와 같은 방향(영어의 경우 왼쪽 -> 오른쪽)을 따라야 하며, 응답의 경우 반대 방향으로**

### TIP
구성 요소의 응답을 **항상 요청과 반대 방향**으로 움직이도록 하기
- 시각적으로 상호작용을 차별화 하면 독자가 다이어그램 더 쉽게 이해 가능

시퀀스 다이어그램은 흐름을 정확히 파악하기 쉬움
- 왼쪽 상단에서 시작하며, 요청은 왼쪽 -> 오른쪽, 응답은 오른쪽 -> 왼쪽
- **흐름이 이어지려면, 상단에 있는 구성 요소의 순서가 중요**

구조 다이어그램도 정보의 흐름이 유사하나, 추가적으로 고려해야 하는 사항 존재
- **데이터베이스와 같은 인프라가 포함된 다이어그램을 만들 때 일반적으로 인프라 요소 배치 후 시스템, 컨테이너 요소 배치**
- 그리고 액터 또는 **사용자와 같은 요소는 맨 위에** 배치하는 것이 일반적
- 이는 절대적인 규칙은 아니나, 이 책에서 제시하는 다른 안티 패턴이 아니라면 지키는 것이 좋음

계층화된 아키텍처를 보여주는 다이어그램과 같은 **논리 다이어그램**은
- 왼쪽 -> 오른쪽, 위 -> 아래 패턴을 따라야 하나, 계층을 고려해야 함
- 계층은 다른 구조 다이어그램과 같이 논리적으로 **위 -> 아래**로 배열해야 하며,
	- 사용자 인터페이스 | API 레이어와 같이 사용자를 대상으로 하는 레이어를 최상단에 배치해야 함
- 계층 안에서 왼쪽 -> 오른쪽 배치하는 가장 논리적인 방법 고려

### NOTE
**육각형 아키텍처 다이어그램**의 경우 레이아웃에 대한 기대치가 약간 다름
- 다이어그램을 시계와 비슷하게 생각
- 왼쪽 상단, 사람들의 시선이 향하는 곳 시작하기 좋으나, 독자의 시선을 중앙으로 집중시킬 수 있음
- 색상. 굵고 큰 텍스트 또는 굵은 선 사용
	- **독자의 시선을 시작 위치로 유도한 다음 논리적으로 시계 방향 배치**
