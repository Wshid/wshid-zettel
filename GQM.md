- GQM(Goal-Question-Metric) 접근법
	- 소프트웨어 조직 전반에 걸쳐 목표 지향적인 측정을 중시하는 방법론
- 이 접근법은 프로젝트의 목표를 간명하게 나타내는 여러 목표들로 설정한 다음,
	- 이러한 목표들을 달성하기 위해 진행 상황을 점검할 수 있는 일련의 지표를 정하는 것
	- 이때 사용하는 지표는 구체적이고, 측정 가능하며, 목표와 연관성이 있어야 함
- 여기에 코드 품질, 프로세스 효율성, 팀 성과와 같은 생산성과 관련된 지표도 포함시킬 수 있음
- 이러한 지표를 사용한다면 비즈니스 성과와 목표를 데이터에 기반한 지표에 연결시켜
	- 프로젝트 전반을 파악할 수 있는 큰 그림을 그릴 수 있게 됨
- 다음과 같은 예시로 주어진 목표에 대해 어떤 지표와 질문을 떠올릴 수 있겠는가
#### 예시: 목표-질문-지표
- 목표
	- 검색 엔진에서 웹 사이트의 순위 올리기
- 질문
	- 웹 사이트의 현재 순위는 어떤가
	- 어떻게 순위를 높일 수 있겠는가
	- 만약, 검색 엔진에서의 순위가 모바일 환경에서의 성능, 사용자 경험에 달려 있다면
		- 다음과 같은 지표 활용 가능
- 지표
	- 모바일 환경에서 페이지 로딩 속도
	- 사용자 경험 지표
